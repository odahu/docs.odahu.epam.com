Legion Platform Architecture¶

Building, training and deploying¶

Legion tries to unify process of training, building and deploying ML/AI models, splitting them to 3 main phases.

  1. Model Training
  2. Model Packaging
  3. Model Deploying

Legion provides an ability to develop applications/tools that can be used for these phases. These applications are:

  1. Model Trainer
  2. Model Packager
  3. Model Deployer

For cloud use case, these applications/tools have to be registered in Legion Platform using:

  1. Toolchain Train Integration
  2. Toolchain Packaging Integration

These registrations are required to have an ability to integrate training / packaging process with other subsystem Legion provides too (e.g. Model Training Metrics, Model Training Tags).

Storing of trained models¶

As an intermediate format for storing trained models Legion Platform declares Trained Model Binary Formats for different languages.

Nowadays, Legion Platform declares:

  1. General Python Prediction Interface

Ready to use solutions¶

There are ready for use Toolchain Train Integrations and Toolchain Packaging Integration:

  1. MLflow Model Training Toolchain Integration
  2. Docker REST API Packaging Toolchain Integration

But Legion Platform users are not limited to set of predefined Toolchain Train and Toolchain Packaging integrations and are free for installation of third-party integrations.

Legion subsystems¶

Legion subsystems are:

  1. EDI
  2. Operator
  3. Feedback aggregator

These subsystems are optional, and can be deployed not just inside Legion Cloud, but even in other products.

Other integrations¶

For integration with Legion, there are libraries and plugins:

  1. Python SDK Library
  2. Legion CLI
  3. Plugin for JupyterLab
  4. Plugin for Jenkins
  5. Plugin for Airflow

Deploying of Legion Platform¶

Legion Platform can be installed locally or on Kubernetes cluster.
